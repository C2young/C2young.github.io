去年参加了一场大型攻防演练，作为蓝队防守方，第一次遇到实战中的攻击排查以及应急响应。因此写一篇博客记录一下，总结应急响应的流程，顺便回顾一下当时接触到的安全设备、流量分析等等。

    尝试漏洞扫描

  5. 时间：2023/08/18 02:33:51

    哥斯拉 webshell 文件上传（上传成功）

  6. 时间：08/18 02:34:14

# 攻击分析排查 / 应急响应（深信服 sip）

  1. 攻击队通过跳板机对受害机器进行弱口令登陆

  2. 通过上机排查发现晚上 2 点 30 分上传哥斯拉 jsp 文件

    对木马文件进行删除处理

  3. 查看分析 web 日志

    发现攻击队上传文件的接口

    后面有持续连接该 webshell 的记录

  4. 查看代码

    发现只通过 js 代码限制类型文件上传，后端接口 thematic/updHomePagePicture.action 没有限制，建议对接口添加文件限制禁止任意文件上传

  5. 查看分析 Windows 安全日志

    在 2 点 34 分后发现通过 3389 登录 administrator 账号登录成功，建议修改密码

  6. 查看用户

    无异常账号添加

  7. 封禁攻击 IP

# 攻击路线

漏洞扫描（无果） => 弱口令登录（成功） => 尝试文件上传（成功） => 上传冰蝎 webshell（文件名：1000kb.jsp）/ 上传哥斯拉 webshell（文件名：1000kbjsp） => 连接哥斯拉 webshell（文件名：202308180233501000kb.jsp） => 上传冰蝎 webshell（文件名：test.jsp） => 连接冰蝎 webshell（文件名：20230818025029test.jsp）

# 修复建议

  1. 弱口令：完善密码策略，密码长度至少为 8（包括）位以上字符，且包含数字、大小写字母、特殊字符。
  2. 及时删除恶意样本 202308180233501000kb.jsp 、20230818025029test.jsp
  3. 文件上传：对文件上传做严格的限制，从后缀名、文件头、文件内容等方面校验。

# Struts2 远程代码执行漏洞

# 攻击发现（微步 TDP）

  1. 时间：2023/08/18 01:55:58

    尝试登录（未成功）

    使用 Struts2 远程代码执行攻击（攻击成功）

# 攻击分析排查 / 应急响应（深信服 sip）

  1. 攻击队通过跳板机对受害机器进行 Struts2 反序列化漏洞，查看返回包发现命令已经成功执行，修复建议升级 Struts2 组件

  2. 查看 web 日志

    发现攻击队在晚上 1 点 57 分发起的攻击

    执行成功后，2 点 02 分后该 IP 没有访问 jsp 文件或者其他路由的内存马，也没有对受害机器进行访问了

  3. 查看定时任务

    没有异常的定时任务行为

  4. 查看 Tmp 目录

    Tmp 目录下没有可疑文件

  5. 封禁攻击 IP

# 攻击路线

尝试登录（无果） => Struts2 远程代码执行攻击（攻击成功）

# 修复建议

  1. 建议将 jar 包更新到最新版本；
  2. struts2 标签库的升级：将原有的 struts-tag.tld 升级为最新。

# velocity 模板注入漏洞

# 攻击发现（微步 TDP）

  1. 时间：2023/08/18 20:14:40

# 攻击分析排查 / 应急响应（深信服 SIP）

  1. SIP 告警 Java 代码注入攻击

  2. 分析流量包

    查看攻击 payload

  3. 解密 BCEL

    得出 class 类，class 类里面用 base64 加载恶意类，通过 base64 解密恶意类，发现哥斯拉连接信息 aeskey 和密码

  4. 清理了攻击者残留的内网扫描工具及结果文件。

  5. 重启 JAVA 应用清理内存 webshell。

  6. 封禁攻击 IP

# 攻击路径

velocity 模板注入攻击（攻击成功）

# 修复建议

升级 velocity 组件版本，对漏洞进行修复，或者通过 WAF 添加临时防护规则对漏洞进行防护。漏洞未修复前或未得到有效的防护前建议不要对互联网开放该应用。

# Nacos 弱口令＋Nacos Jraft Hessian 反序列化攻击

# 攻击发现（微步 TDP）

  1. 时间：2023/08/19 01:33:16

    nacos 弱口令登陆

    弱口令：nacos/nacos

  2. 时间：2023/08/19 02:09:45

    Nacos Jraft Hessian 反序列化攻击

  3. 时间：2023/08/19 02:11:43 - 2023/08/19 02:30:35

    哥斯拉内存马通信

# 攻击分析排查 / 应急响应

登录受害主机，识别可疑进程为 file:// Content 已删除进程。系统日志，仅保留到 22 年，无最新日志。
访问 nacos 业务系统，看到平台存在一些可执行配置文件，协调同事排查哪些属于正常业务系统的配置文件，确认完成后，清除可疑文件，另外协同事，部署终端主机查杀，做一次安全巡检。当前主机为断网隔离状态。

# 总结

# 准备阶段

  1. 系统内自查，尽可能消除弱口令，做好宣传动员，提升防守意识，防止钓鱼

  2. 部署安全设备，如态势感知（微步 TDP / 深信服 SIP / 奇安信 NGSOC 等等），防火墙（新华三防火墙 / 迪普防火墙），waf（长亭雷池），蜜罐（长亭谛听 / 默安幻阵 / 观安蜜网），堡垒机（安恒明御），全流量分析（科来全流量安全分析系统），负载均衡（新华三负载均衡），主机防护（青藤万相）......

  3. 先使用漏洞扫描系统（绿盟漏扫）对资产进行一次全面扫描自查

# 攻击发现阶段

  1. 通过态势感知产品（微步 TDP / 深信服 SIP / 奇安信 NGSOC 等等）的告警发现攻击（大部分的入侵点为弱口令以及 nday）

  2. 分析告警信息以及流量包，确认攻击是否成功，攻击阶段是否已经入侵内网 / 是否已经扩散，攻击者是否上传 webshell 或内存马等等（看攻击 IP、受害 IP 是否内网 IP，若是，则可能内网失陷；看响应码是否为 200，响应内容与请求内容是否相应，若相应，则可能攻击成功）

# 攻击分析排查 / 应急响应阶段

  1. 封禁攻击 IP

  2. 受害主机进行断网隔离

  3. 上机排查

    • 查看 web 日志，分析攻击者主要进行了什么活动（是否上传 webshell / 是否存在内存马）

    • 若有 webshell，找出 webshell 上传接口，上传路径，webshell 文件名

      • 确认 webshell 位置，清理木马文件以及其他可疑文件

      • D 盾防火墙查杀

      • 查看代码，及时修复上传漏洞

    • 若有内存马，使用工具（java-memshell-scanner/copagent/shell-analyzer）查杀

      • 识别可疑进程，终止进程（Windows：tasklist /svc；Linux：top、ps、pstree）

      • 直接重启服务（在确认不影响客户正常业务，征得客户同意后）

  4. 查看 Tmp 目录，是否存在可疑文件，及时清理

  5. 查看分析安全日志，攻击者是否登陆 administrator 用户 / 添加新用户

  6. net user 查看是否添加异常用户

  7. crontab -l 查看是否存在异常定时任务

  8. 部署主机防护系统（青藤万相）查杀，进行安全巡检

  9. 给出修复建议 / 修复漏洞，重新上线服务

# 一些应急响应过程中需要用到的命令

  1. Windows 安全日志位于事件查看器 / Windows 日志 / 安全（Windows id 时间响应码）

    • 4624 登录成功

    • 4625 登录失败

    • 4672 使用超级管理员进行登录

    • 4720 创建用户

  2. 查看用户

    • Windows：net uer

    • Linux：cat /etc/passwd（文件后面第二个冒号的值大于 1000 时，这个就是一个用户）

  3. 查看进程

    • Windows：tasklist /svc

    • Linux：top、ps、pstree

  4. 查看定时任务

    • Windows：taskschd.msc

    • Linux：crontab -l

# 结束阶段
