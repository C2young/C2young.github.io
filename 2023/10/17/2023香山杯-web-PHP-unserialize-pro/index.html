源码如下

观察代码，最后应该是需要进到 G00d类 里面的 __invoke() 来执行 eval 命令，触发 __invoke() 需要把这个类当作函数来调用执行，所以需要进入到 H4ck3r类 里面的 __tostring() ，触发 __tostring() 需要把这个类当作字符串来调用，刚好 welcome类 里面的 echo $this->arg; 需要调用字符串，而其所在的 __destruct() 是类实例化后销毁时会自动触发的。至此，完整的调用链为 welcome类__destruct() -> H4ck3r类__tostring() -> G00d类__invoke() 。

然后接下来就是绕过 preg_match () 的问题了

一开始想的有三种方案：1、替代函数；2、取反绕过；3、正则匹配绕过（因为通配符也被禁了所以无法使用通配符绕过）

  1. 替代函数

    使用 more、sort 可以用于 cat 被过滤的情况；dir、vdir 可以替代 ls

    但是因为这个过滤是一个个字母的匹配且过滤了通配符，所以绕过了 cat 的限制后还是没法读取 flag

  2. 取反绕过

    这个方法也没成功，但是我一直在纠结为什么不能取反，在这花费了很多时间，所以先把这个坑摆上来，弄懂了再来填

    对字母取反

    最后测出来是空格过不了过滤（为什么呢？）于是又想到命令拼接的形式，就是对 cat 和 /flag 取反，然后中间的空格直接通过。拼接

    但是这个方法最终也没有成功（为什么呢？）经过 unknown 的指点，明白了。因为传过去的 cmd 不会自己解码，他是一整个字符串传过去给 system 执行的，相当于直接执行了 '~%9C%9E%8B'.' '.'% D0%99%93%9E%98'，所以没法执行。想要执行除非用 system (urldecode ('%9C%9E%8B'.' '.'~% D0%99%93%9E%98')，更加麻烦。

  3. 正则匹配绕过

    这种方法通过 [x-y] 匹配 x 到 y 之间的字符（一开始是想到这个的，但是由于匹配的范围太广不够精准，估计匹配到了 /home，然后回显报错权限不足了，以为是这个方法行不通，所以又浪费了很多时间，悲）

    后看大佬的 wp 还有很多中绕过方式，例如 chr () 绕过等等，都摆出来放在下面，积累吧

这就是一道比较简单的 php 反序列化题，其中比较难的是绕过 preg_match ()。
